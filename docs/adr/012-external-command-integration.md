# ADR-012: 外部コマンド連携による暗号化/復号化機能

## Status

Accepted

## Decision

暗号化/復号化機能を **完全外部化パターン（P4）** で実装する。Gyaimはトリガー検出・外部コマンド実行・候補表示のみを担当し、暗号化ロジック・認証・保管はすべてユーザーが用意する外部スクリプト（`~/.gyaim/secret.sh`）に委ねる。

## Context

Ruby版GyaimMotionには `?` サフィックスで暗号化/復号化する機能があったが、ADR-002で削除された（`?`が通常入力を潰す、暗黙的トリガーのUX問題）。

ユーザーはKeychain、EpisoPass、1Password CLI、GPGなど多様な暗号化手段を使い分けたいという要望があり、特定の暗号化方式をハードコードするのではなく、すべてのパターンを包含する外部コマンド方式を採用した。

## Consideration

| パターン | 概要 | 評価 |
|---------|------|------|
| P1: Keychain直接 | Gyaim内でSecurity.frameworkを直接呼ぶ | Keychainに限定される |
| P2: EpisoPass組込 | EpisoPassのMD5暗号をGyaim内に実装 | 特定方式に依存 |
| P3: プラグイン | Swiftプラグイン機構を設計 | 過剰な複雑さ |
| **P4: 完全外部化** | シェルスクリプトに全委任 | **全パターンを包含** |

P4はP1〜P3のすべてを包含する。ユーザーがスクリプト内でKeychain、EpisoPass、1Password CLI等を自由に組み合わせ可能。

## Design

### 外部コマンド規約

```bash
~/.gyaim/secret.sh <subcommand> [argument]
```

| サブコマンド | 引数 | stdout | 用途 |
|-------------|------|--------|------|
| `encrypt` | 平文 | メッセージ（1行=1候補） | 暗号化+保存 |
| `list` | なし | ラベル名（1行=1ラベル） | 復号可能な項目一覧 |
| `decrypt` | ラベル名 | 復号済み平文（1行=1候補） | 復号化 |

### トリガー方式（両方対応）

- **接尾辞トリガー**: `平文@e` → 暗号化、`@d` → 復号化一覧（トリガー文字は設定変更可能）
- **ショートカット**: 設定画面で任意のキーバインドを追加

### 復号化の2段階フロー

1. `secret.sh list` → ラベル一覧を候補表示
2. ユーザーがラベルを選択・確定 → `secret.sh decrypt "ラベル"` → 復号結果を候補表示 → 確定で挿入

2段階目の検出は `SearchCandidate.reading` に `@decrypt:ラベル` プレフィックスを付与し、`fix()` で識別する。

### アーキテクチャ

Google Transliterateと同一パターン: 非同期Process実行 → stale guard → メインスレッドで候補更新。待機中は「(実行中...)」を候補表示。タイムアウト5秒。

### IMEリセット

外部コマンドがGUIを表示する可能性があるため、コマンド実行前にGyaimのIME状態をリセットする（inputPat空、マーク解除、候補ウィンドウ非表示）。これにより外部GUIのテキスト入力をGyaimが横取りしない。encrypt / list / decrypt の3サブコマンドすべてで共通。

### 将来の拡張: EpisoPass連携

P4アーキテクチャはEpisoPass（エピソード記憶ベースの暗号鍵導出）との連携を想定している。実現方法は2つ:

**方式A: EpisoPass Web + クリップボード中継**

1. secret.sh が `open https://episopass.com/?data=...` でブラウザを開く
2. ユーザーがWeb上でエピソード質問に回答 → 鍵が生成される
3. ユーザーが鍵をコピー → secret.sh が `pbpaste` で受け取る
4. 鍵で暗号化/復号化してstdoutに出力

課題: ブラウザ操作完了をスクリプトが検知するタイミング制御が必要（ポーリングまたはローカルHTTPサーバーでコールバック受信）。

**方式B: EpisoPass CLIツール（推奨）**

1. SwiftでローカルCLIツールを作成（Cocoaウィンドウで質問表示 → 回答 → MD5ハッシュで鍵導出 → stdoutに出力）
2. secret.sh から `KEY=$(episopass-cli --questions ~/.gyaim/questions.json)` で呼び出す
3. 鍵で暗号化/復号化してstdoutに出力

利点: IMEリセット済みのためCocoaウィンドウへの入力が妨害されない。ブラウザ不要でオフライン動作可能。スクリプトとのstdout連携がシンプル。

いずれの方式もsecret.shの差し替えのみで実現でき、Gyaim本体の変更は不要。

## Consequences

### 良い点

- ユーザーが暗号化手段を自由に選択・組み合わせ可能
- Gyaim本体は暗号化ロジックを一切持たない（責務分離）
- ADR-002で削除された `?` トリガーの問題を回避（`@e`/`@d` は設定変更可能）
- Google Transliterateと同じアーキテクチャパターンで保守しやすい

### 悪い点

- ユーザーがスクリプトを自分で書く/編集する必要がある（サンプルスクリプトで緩和）
- シェル経由の実行はセキュリティリスクがある（シェルエスケープ + タイムアウトで緩和）
- 外部プロセスの起動コストがある（暗号化/復号化の頻度は低いため許容範囲）
